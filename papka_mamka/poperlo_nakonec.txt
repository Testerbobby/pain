Полезные советы по коммитам
Пишите сообщения кратко и по делу, первая строка до 50 символов, в повелительном наклонении: «Add tests», «Fix login bug».​

Формируйте логически атомарные коммиты: сначала выбирайте изменения через git add <file> или git add -p, затем фиксируйте их в один осмысленный коммит.​

Всегда проверяйте git status перед коммитом, чтобы убедиться, что в индексе именно то, что хотите зафиксировать.​

Если push не проходит
Чаще всего локальная ветка отстаёт: сделайте git pull --rebase origin main, решите конфликты, затем git push.​

Убедитесь, что удалённый URL верный и у вас настроена аутентификация (HTTPS токен или SSH ключ).​
